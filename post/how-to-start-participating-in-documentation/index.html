How to start participating in documentation？

2023/03/23 2 min read
  1. Register an account at Github. (Optional: to get the benefits of the GitHub Student Developer Pack along with it, please sign up with your BHCC email address)
  2. Send me a private message with the email address that linked to your github accout to get access to the private repository.
  3. Download any text editor, since we're using markdown as mark up language, I recommand MarkText(VS Code is fine, it has a better ability to work with Github Desktop (in terms of being able to launch files with one click and show differences faster), however I think it has a better out-of-the-box and specialized experience as Marktext).
  4. (For pros: If you know how to use git, then use git) Download Github Desktop, follow the in app tutorial to get the basic of pull, push and commit.
  5. Create a new working branch, named as what you are going to work in
  6. Pull the document repo to local, set editor work directory to that local directory.
  7. Start to write.
  8. Push and Commit.
  9. When you think it is ready to merge to main branch, for example, a stage has been done, merge it to main branch.
